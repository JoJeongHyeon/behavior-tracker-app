# 가입과 로그인

현재 1차 버전은 이메일/비밀번호 가입과 로그인을 기준으로 합니다.

## 현재 방식

- 이메일로 회원가입
- 비밀번호로 로그인
- Supabase Auth 사용
- 로그인한 사용자 본인 기록만 볼 수 있음
- 인증 메일 링크는 실제 배포 웹사이트로 이동함
- 가입 요청 중 로딩 상태 표시
- 인증 메일 발송 후 확인 화면 표시
- 인증 메일 다시 보내기 제공
- 이메일 미인증, 비밀번호 오류, 과도한 요청 같은 오류를 한국어로 안내

## 지금 사용자가 확인할 것

1. 배포 주소를 연다.
2. 이메일과 비밀번호로 가입한다.
3. `인증 메일을 보냈습니다` 화면이 나오는지 확인한다.
4. 이메일 인증 메일이 오면 인증한다.
5. 다시 로그인한다.
6. 오늘 기록을 저장한다.
7. 로그아웃 후 다시 로그인해 기록이 남아 있는지 확인한다.

## 인증 메일이 안 올 때 확인할 것

사용자 화면에서 먼저 확인할 것:

- 이메일 주소에 오타가 없는지 확인한다.
- 스팸함, 프로모션함, 전체 메일함을 확인한다.
- 1분 정도 기다린 뒤 `다시 보내기`를 누른다.
- 이미 가입한 이메일이면 로그인으로 전환해본다.

관리자 화면에서 확인할 것:

- Supabase Dashboard에서 Authentication 설정을 확인한다.
- 이메일 회원가입이 켜져 있는지 확인한다.
- 이메일 인증이 켜져 있다면 메일 발송 로그를 확인한다.
- Site URL과 Redirect URL에 배포 주소가 들어가 있는지 확인한다.
- 앱의 `NEXT_PUBLIC_SITE_URL` 값이 배포 주소인지 확인한다.
- 기본 메일 발송 한도나 지연이 의심되면 SMTP 설정을 검토한다.

## 가입 화면 UX 기준

- 가입 버튼을 누르면 바로 반응이 보여야 한다.
- 메일이 전송됐는지 사용자가 분명히 알아야 한다.
- 다음 행동이 화면에 남아 있어야 한다.
- 오류가 나면 영어 원문보다 짧은 한국어 안내를 보여준다.
- 가입 화면은 불안하지 않고 차분한 서비스 앱처럼 보여야 한다.

## 다음에 추가할 소셜 로그인

아래 로그인 방식을 추가할 예정입니다.

- 구글 로그인
- 네이버 로그인
- 카카오 로그인

## 소셜 로그인을 추가하면 좋아지는 점

- 비밀번호를 따로 만들지 않아도 된다.
- 모바일에서 가입 과정이 짧아진다.
- 매일 기록하는 앱에 다시 들어오기 편해진다.
- 이메일 인증 과정에서 막히는 사용자가 줄어든다.

## 구현할 때 확인할 것

- Supabase Auth Provider에서 Google, Kakao, Naver 설정이 가능한지 확인한다.
- 각 서비스의 개발자 콘솔에서 OAuth 앱을 만든다.
- Redirect URL을 Supabase와 각 서비스 콘솔에 정확히 등록한다.
- Vercel 배포 주소와 로컬 주소가 모두 로그인 리다이렉트에 포함되는지 확인한다.
- 같은 이메일로 가입한 계정이 중복 사용자처럼 보이지 않는지 확인한다.

## 우선순위 제안

1. 구글 로그인
2. 카카오 로그인
3. 네이버 로그인

구글은 설정과 테스트가 비교적 쉽고, 카카오는 한국 사용자에게 익숙합니다. 네이버는 필요하지만 OAuth 설정이 조금 더 번거로울 수 있어 세 번째로 둡니다.
