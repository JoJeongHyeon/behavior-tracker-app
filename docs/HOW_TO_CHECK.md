# 직접 확인하는 방법

개발을 몰라도 아래 순서대로 눌러보면 현재 앱이 어디까지 되는지 확인할 수 있습니다.

## 1. 앱 열기

- 배포 주소: https://behavior-tracker-app-khaki.vercel.app
- 로컬 주소: http://localhost:3000

배포 주소는 휴대폰에서도 열 수 있습니다. 로컬 주소는 지금 이 컴퓨터에서만 열립니다.

## 2. 회원가입과 로그인 확인

현재는 이메일/비밀번호 가입을 확인합니다. 구글, 카카오, 네이버 로그인은 다음 업데이트 후보입니다. 가입과 로그인 계획은 [AUTH.md](AUTH.md)에 정리되어 있습니다.

1. 배포 주소를 연다.
2. 이메일과 비밀번호를 입력한다.
3. 가입을 누른다.
4. 버튼이 `처리 중` 상태로 바뀌는지 확인한다.
5. `인증 메일을 보냈어` 화면이 나오는지 확인한다.
6. 메일이 안 오면 `다시 보내기`를 눌러본다.
7. 이메일 인증 메일이 오면 인증 링크를 누른다.
8. 다시 앱에 들어와 로그인한다.

확인할 것:

- 로그인 화면 문구가 이해되는가?
- 실패했을 때 메시지가 너무 어렵지 않은가?
- 이메일 인증 과정이 헷갈리지 않는가?
- 메일이 안 올 때 사용자가 무엇을 해야 하는지 화면에 보이는가?
- 인증 메일 다시 보내기 버튼이 찾기 쉬운가?

## 3. 오늘 기록 확인

아래 항목을 하나씩 눌러본다.

- 몸무게
- 식단
- 운동
- 공부
- 시쓰기
- 책읽기

확인할 것:

- `예`를 누르면 입력칸이 자연스럽게 열리는가?
- `아니오`를 누르면 입력칸이 닫히는가?
- 모바일에서 버튼이 누르기 편한가?
- 저장 버튼이 찾기 쉬운가?

## 4. 식단 점수 확인

아래 경우를 테스트한다.

- 간식과 야식을 모두 안 먹음: `계속 그렇게 가보자`
- 간식은 먹고 야식은 안 먹음: `야식은 잘 넘겼어`
- 간식은 안 먹고 야식은 먹음: `간식은 잘 참았어`
- 간식과 야식을 먹었지만 기록함: `기록한 건 좋았어`

확인할 것:

- 문구가 짧고 자연스러운가?
- 너무 AI처럼 느껴지지 않는가?
- 칭찬이 부담스럽거나 과하지 않은가?

## 5. 운동 기록 확인

테스트 예시:

- 걷기: 40분
- 팔굽혀펴기: 30회
- 스쿼트: 50회
- 플랭크: 3세트

확인할 것:

- 시간과 횟수를 둘 다 입력할 수 있는가?
- 운동을 여러 개 추가할 수 있는가?
- 삭제 버튼이 헷갈리지 않는가?

## 6. 독서 계산 확인

테스트 예시:

- 시작 페이지: 10
- 끝 페이지: 35
- 총 페이지: 100

기대 결과:

- 읽은 쪽수: 26쪽
- 진행률: 35%

## 7. 기록 목록 확인

1. 오늘 기록을 저장한다.
2. `기록` 탭을 누른다.
3. 저장한 날짜의 카드가 보이는지 확인한다.
4. `수정` 버튼을 눌러 오늘 기록 화면으로 돌아가는지 확인한다.

## 8. 월간 요약 확인

1. `요약` 탭을 누른다.
2. 기록일, 운동 시간, 공부 시간, 독서량이 바뀌는지 확인한다.

확인할 것:

- 숫자가 이해하기 쉬운가?
- 더 보고 싶은 요약이 있는가?
- 그래프가 있으면 좋겠는 항목이 있는가?

## 9. 모바일 확인

휴대폰에서 배포 주소를 연다.

확인할 것:

- 한 손으로 기록하기 편한가?
- 글자가 작거나 답답하지 않은가?
- 저장 버튼이 잘 보이는가?
- 애니메이션이 보기 좋은가, 산만한가?

## 10. 피드백 남길 때 보면 좋은 것

피드백을 줄 때는 아래처럼 말하면 바로 개선하기 좋습니다.

- 어느 화면인지
- 어떤 행동을 했는지
- 기대한 결과
- 실제 결과
- 불편하거나 어색했던 문구
- 모바일인지 PC인지

예시:

```text
모바일에서 식단 기록할 때 간식 입력칸이 조금 좁아 보여.
간식 안 먹음을 선택했을 때 문구는 좋은데, 저장 후 더 눈에 띄면 좋겠어.
```
